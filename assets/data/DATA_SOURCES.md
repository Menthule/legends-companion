# Bundled reference data

## drops.sqlite

Item → mob → zone drop-research database used by the app's **Drops** tab.

- **Source:** [ProjectEQ (PEQ)](https://sourceforge.net/projects/projecteq/)
  Luclin-era database release (`peq-luclin-release.zip`, 2006-11-30), the
  community-maintained content database of the open-source
  [EQEmu](https://github.com/EQEmu/Server) EverQuest emulator project.
- **Generated by:** `tools/dropdata/build_drops_db.py` (run it against the
  extracted `load_system.sql` to reproduce this file byte-for-byte apart
  from row order).
- **What it is:** classic-era *reference* data — item names, stats, which
  NPCs drop them at what approximate chance, and where those NPCs spawn,
  with each zone tagged by era (classic / Kunark / Velious / later).
- **What it is not:** EverQuest Legends' actual loot tables. Legends
  re-itemizes gear (merged +N upgrades, augments, changed stats); treat
  everything here as a guide to *where to hunt*, not as exact numbers.

### vendor / recipe / zone research tables

Additional reference tables built from the same PEQ dump by
`build_drops_db.py`: `vendor_items` (which kept NPC sells which item, via
`merchantlist`; merchant NPCs are kept even when they drop nothing),
`recipes` + `recipe_components` + `recipe_results` (tradeskill recipes
with trivial and no-fail flags), `zone_forage` and `zone_fishing`
(forageable/fishable items per zone short name; rows keyed to no known
zone — e.g. the global fishing table — are dropped), and
`zone_connections` (deduped zone-to-zone links from `zone_points`).
The `items` table gains `scroll_spell_id` (spell taught by a learnable
scroll, 0 otherwise), `npcs` gains `named` (raw name began with `#`),
`merchant_id`, and `faction` (primary faction name, NULL if none), and
`npc_zones` gains `respawn_secs` (average spawn-point respawn time for
that NPC in that zone). Same era, same caveats: classic-era emulator
reference data, not Legends' live values.

### spells + spell_classes tables

Spell/ability reference tables appended into the same `drops.sqlite`.

- **Source:** the EverQuest Legends client's own spell files
  (`spells_us.txt`, caret-delimited, plus `spells_us_str.txt` for the
  cast/wear-off log messages), kept locally at `fixtures/local/`.
- **Generated by:** `tools/spelldata/build_spell_db.py` (run after
  `build_drops_db.py`; idempotent — it drops and recreates only its own
  two tables).
- **What it is:** every spell or combat ability castable by one of the
  16 classes at level ≤ 60 — mana/endurance cost, cast/recast times,
  a level-50 duration estimate (seconds; 432000 = until dispelled),
  range, target/resist type, casting skill, per-class levels, and the
  exact log-message strings. `is_ability = 1` marks endurance-based
  disciplines/combat abilities (endurance cost or upkeep > 0, or the
  client's discipline flag set); `0` is a mana spell (bard songs
  included). Rank ("Rk. II") copies and placeholder rows are excluded.
- **What it is not:** guaranteed to match Legends' live tuning after
  patches — regenerate from a fresh client file when the game updates.
  Note Legends diverges from classic (e.g. no player-castable
  Complete Heal exists in the file).

## Distribution / updates

Installed apps receive updated copies of this database (plus the trigger
packs) through the rolling `data-latest` GitHub release channel
(Settings → Updates in the app). `tools/release/make_data_pack.py` builds
that pack — drops.sqlite, triggers.zip, and a sha256 `data-manifest.json` —
deterministically from the current repo contents.
